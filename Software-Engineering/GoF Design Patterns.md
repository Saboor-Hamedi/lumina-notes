# GoF Design Patterns

Gang of Four patterns are reusable solutions to common software design problems.

## Creational

| Pattern | Purpose | Example |
|---------|---------|---------|
| Singleton | One global instance | Logger, config |
| Factory | Create objects without specifying concrete class | DB connection factory |
| Builder | Step-by-step complex object construction | SQL query builder |
| Prototype | Clone existing objects | Object caching |

## Structural

| Pattern | Purpose | Example |
|---------|---------|---------|
| Adapter | Match incompatible interfaces | Legacy system wrapper |
| Decorator | Add behavior dynamically | Middleware, pipeline |
| Proxy | Control access to an object | Lazy loading, caching |
| Facade | Simplified interface to complex subsystem | API gateway |

## Behavioral

| Pattern | Purpose | Example |
|---------|---------|---------|
| Observer | One-to-many notification | Event listeners |
| Strategy | Swappable algorithms | Sort strategy, payment method |
| Command | Encapsulate request as object | Undo/redo, task queue |
| Iterator | Sequential access to collection | for-each loops |

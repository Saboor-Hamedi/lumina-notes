# Mission: PostgreSQL Mastery

## Why
Master PostgreSQL at the level where I can design, tune, and troubleshoot production databases with confidence — whether building full-stack apps, moving into a backend/data role, or optimizing existing systems. I want to move beyond "it works" to "I know exactly why it works this way."

## Success looks like
- Read an `EXPLAIN ANALYZE` plan and immediately identify the bottleneck
- Design normalized schemas that handle real-world OLTP workloads without pain
- Diagnose and fix performance issues (slow queries, bloat, connection storms) in production
- Configure PostgreSQL for high availability with replication and failover
- Choose the right indexing strategy (B-tree, GiST, GIN, BRIN) for any query pattern
- Deploy extensions (pgvector, PostGIS) and tune them for the workload

## Constraints
- Self-paced — no fixed deadline, depth over speed
- Learning through the existing Obsidian vault as reference material
- Hands-on exercises preferred over theory-only

## Out of scope
- NoSQL systems (MongoDB, Cassandra) — focused on PostgreSQL / relational mastery
- ORM-specific knowledge (this is about the database, not the abstraction layer)
- Cloud-specific managed services (RDS, Aurora) — focus on core PostgreSQL first

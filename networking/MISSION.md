# Mission: Networking & Protocols

## Why
Understand how the internet works — from HTTP requests to DNS resolution to TCP handshakes. Essential for debugging, security, and system design.

## Success looks like
- Explain what happens when you type a URL into a browser
- Debug connectivity issues using ping, traceroute, curl
- Understand TCP, UDP, HTTP/1.1, HTTP/2, TLS
- Read and interpret packet captures (Wireshark basics)
- Design network layouts for applications

## Constraints
- Self-paced, hands-on
- Familiar with basic Linux commands (curl, ping)
- Mostly theoretical with practical exercises

## Out of scope
- Network engineering / Cisco certification
- Cloud networking (VPC, subnets) — focus on fundamentals

# Mission: Linux Command Line Mastery

## Why
Master the Linux command line for server management and DevOps workflows — deploying apps, managing infrastructure, automating tasks, and troubleshooting production systems. I want to go from "I can get around" to "I can automate anything."

## Success looks like
- Navigate, manage files, and edit configs on any Linux server without hesitation
- Write shell scripts to automate repetitive tasks (backups, deployments, log rotation)
- Understand process management, permissions, and system administration basics
- Use text processing tools (grep, sed, awk) for log analysis and data extraction
- Debug production issues: check logs, resource usage, network connectivity
- Manage services, users, and packages on Linux servers

## Constraints
- Running on Windows — will use WSL or Docker for practice
- Self-paced, basic → advanced progression
- Hands-on exercises > theory

## Out of scope
- Linux kernel internals or driver development
- Desktop Linux / GUI tools
- Cloud-specific CLI tools (awscli, gcloud) — focus on core Linux first
